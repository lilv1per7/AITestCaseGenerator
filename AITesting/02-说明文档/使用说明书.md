# 使用说明书：AI 测试用例生成器

## 1. 适用场景
**这个工具解决什么问题？**
在软件开发生命周期中，测试人员和开发人员往往需要耗费大量时间去阅读需求文档并手动编写测试用例。该过程容易出现边界值遗漏、异常场景覆盖不全的问题。本工具通过接入大语言模型（DeepSeek-V3），只需输入简要的自然语言需求，即可自动运用等价类划分、边界值分析等专业测试理论，秒级生成结构化、高覆盖率的测试用例，并支持一键导出为 Excel/Markdown。

**谁应该使用它？**
- **测试工程师 (QA)**：快速生成测试基线，提升用例编写效率，查漏补缺。
- **开发工程师 (RD)**：在编码前或提测前，通过工具快速明确需要覆盖的测试场景，指导代码开发（TDD思想）。
- **产品经理 (PM)**：验证需求文档的严谨性，确保验收标准的完整度。

## 2. 快速开始

### 安装与访问步骤
本项目为纯前端应用（React + Vite），无需配置复杂后端数据库。

1. **环境准备**：请确保您的电脑已安装 Node.js (v18+) 及 npm。
2. **进入项目目录**：
   ```bash
   cd 01-交付作品/ai-test-case-generator
   ```

3. **安装依赖**：
   ```bash
   npm install
   ```
4. **配置 API Key（必做项）**：
   在项目根目录下的 `.env` 文件，填入您的API Key：
   ```env
   VITE_DEEPSEEK_API_KEY=sk-您的实际API密钥
   VITE_DEEPSEEK_MODEL=deepseek-ai/DeepSeek-V3
   ```

5. **启动项目**：

   ```bash
   npm run dev
   ```
   启动成功后，浏览器访问终端中提示的地址（默认为 `http://localhost:5173`）。

### 一个简单示例
1. 打开浏览器进入系统首页，点击 开始生成。
2. 在需求录入框中输入：“用户登录功能。用户名3-20位字母数字下划线，密码6-20位。连续5次登录失败锁定30分钟。”
3. 在下方配置项中，选择测试类型偏好（全部/功能/边界/异常）为 “全部”，期望数量调整为 15。
4. 点击 生成用例 按钮，页面将像打字机一样实时流式渲染出生成的测试用例卡片。
5. 点击"导出 Excel"或"导出 Markdown"下载文档，即可获得结构化报表。


## 3. 功能说明

| 功能      | 说明                               |
| ------- | -------------------------------- |
| AI 智能生成 | 基于 DeepSeek-V3 大模型，运用测试方法论自动生成用例 |
| 需求录入    | 支持文本粘贴方式输入需求文档                   |
| 用例展示    | 卡片式列表，支持展开查看详情、筛选、搜索             |
| 用例编辑    | 支持在线编辑用例字段，调整优先级和类型              |
| 多格式导出   | Excel（含统计摘要）和 Markdown 两种格式      |


- **AI 智能解析与流式生成**：不仅基于语义理解生成用例，且采用流式响应（Streaming）技术，用户无需面对漫长的“加载中”白屏，所见即所得。
- **多维度条件筛选**：
  - 支持按用例标题或所属模块进行**关键字实时搜索**。
  - 支持按用例优先级（P0/P1/P2/P3）进行**快速分类过滤**。
- **动态卡片视图**：
  - 测试用例以卡片形式展示，包含优先级徽章、自动化建议标签。
  - 点击卡片即可展开查看详细的前置条件、有序测试步骤、预期结果以及测试数据。
- **双格式纯前端导出**：
  - **Excel 导出**：自动设置列宽、支持换行，且附带一份根据当前用例自动计算的“汇总统计”报表。
  - **Markdown 导出**：标准 Markdown 表格输出，适合一键复制粘贴至 Jira、Confluence 等团队协作平台。
  - **筛选联动**：导出的内容永远与您当前页面筛选/搜索出的结果保持同步。

## 4. 常见问题

**Q: 为什么点击生成后弹出了“AI 生成失败，将展示 Mock 数据”的提示？**
A: 通常是因为您尚未在 `.env` 文件中配置正确的 `VITE_DEEPSEEK_API_KEY`，或者您的网络环境无法访问 `api.siliconflow.cn`。请检查配置并重启 `npm run dev`。为了保证您的体验，系统在报错时会自动降级为您展示 5 条高质量的内置模拟数据。

**Q: 为什么感觉生成的过程还是有点慢？**
A: 由于我们要求大模型返回极为严格的 JSON 结构，并且每个用例都包含步骤、结果等多项长文本字段，消耗的 Token 较大。目前的流式输出已经让等待过程“可视化”，请耐心观察打字效果。

**Q: 导出的 Excel 为什么“操作步骤”没有换行？**
A: 工具在导出 Excel 时已使用 `\n` 进行换行处理，如果您使用的部分老旧表格软件默认未开启“自动换行”功能，请全选该列并点击“自动换行”按钮即可完美呈现。
