# 技术实现说明书：AI 测试用例生成器

## 1. 整体架构

### 技术选型及理由
本项目坚持轻量化与高可用性，选用了现代化的纯前端技术栈：
- **核心框架**：React 18 + TypeScript。组件化易于维护，强类型提升了 JSON 结构解析和业务逻辑的稳定性。
- **构建工具**：Vite。提供极速的冷启动和 HMR（热更新），且其 proxy 机制完美解决了纯前端直接调用大模型接口时的跨域难题。
- **UI 库**：Tailwind CSS v3。无需引入庞大的组件库，通过原子化 CSS 高效构建深色沉浸式主题，且构建产物体积极小。
- **导出方案**：`xlsx` 库生成 Excel，原生 `Blob` 生成 Markdown。不依赖服务端资源流，实现全离线式数据导出。

## 2. 关键实现

### 2.1 模块功能描述
- **路由模块 (`App.tsx`)**：控制首页（Dashboard）与工具页（RequirementInput）的跳转与鉴权拦截。
- **状态管理模块**：利用 `useState` 和 `Set` 结构实现多条件实时过滤与手风琴卡片的无感展开。
- **AI 交互模块 (`aiService.ts`)**：核心层。负责发送请求并处理 SSE 流式数据。
- **纯前端导出模块 (`exportUtils.ts`)**：将 JSON 数据重组、映射并封装为文件流下载。

### 2.2 核心代码片段：流式 JSON 解析与容错
大模型在流式输出（Streaming）时，前端接收到的是不完整的 JSON 碎片。为了实现“打字机”式的实时卡片渲染，我们设计了**截断补全算法**：

```typescript
// src/services/aiService.ts 节选
let validJsonToParse = jsonString;

// 检查当前接收的 JSON 是否已闭合
if (!validJsonToParse.endsWith('}')) {
   // 寻找最后一个有效对象的闭合处
   const lastBraceIndex = validJsonToParse.lastIndexOf('}');
   if (lastBraceIndex !== -1) {
     // 强制补全 JSON 数组和对象的结尾，使其可被 JSON.parse 解析
     validJsonToParse = validJsonToParse.substring(0, lastBraceIndex + 1) + ']}';
   } else {
     continue; // 若连一个完整的对象都没生成完，则跳过本次渲染
   }
}

const parsedObj = JSON.parse(validJsonToParse);
// 过滤掉还未生成完 id 和 title 的残缺用例，避免页面闪烁
const validCases = parsedObj.testCases.filter((tc: any) => tc.id && tc.title);
onUpdate(validCases as TestCase[]);
```

### 2.3 关键决策说明
- **为什么不使用 Node/Python 后端？**
  纯前端工具分发门槛最低。通过配置 Vite 的 `server.proxy` 配合 `secure: false`（跳过本地 TLS 校验），成功在开发环境下实现了绕过浏览器 CORS 限制直连大语言模型 API。
- **为什么选择流式（Streaming）输出而不是一次性等待？**
  结构化的测试用例 Token 消耗极大，一次性生成通常需要等待 1 分钟左右，极易造成“系统卡死”的错觉。流式输出能在 2 秒内给出首字响应，极大优化了用户体验。

## 3. 使用的 AI 工具

在本项目的设计、开发及排错环节，重度依赖了以下 AI 协作工具：
- **Trae IDE**：作为核心开发环境，提供了基于 Agent 的全自动脚手架生成、文件目录维护以及错误排查。例如，在遇到 `UNABLE_TO_GET_ISSUER_CERT_LOCALLY`（TLS 证书报错）时，由 Trae Agent 自动排查定位，并修改了 Vite 配置中的 `secure: false`。
- **DeepSeek-V3 (SiliconFlow)**：作为本项目的核心驱动引擎。在运行时负责根据用户的简要自然语言，遵循内置的测试理论 Prompt（等价类、边界值），进行严密的逻辑推理并吐出结构化的测试用例 JSON。
