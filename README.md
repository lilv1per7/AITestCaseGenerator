<img width="1013" height="430" alt="image" src="https://github.com/user-attachments/assets/f5b78e86-9005-4a8c-8c72-c4b711c257c0" /># AI 测试用例生成器项目说明

本项目为“测试开发（实习）”场景下的交付作品，核心目标是将自然语言需求快速转换为结构化测试用例，降低手工编写用例的时间成本，提升边界值、异常场景等测试覆盖率。

项目主体为一个基于 React + TypeScript + Vite 实现的纯前端应用，支持调用大模型生成测试用例，并导出为 Excel 或 Markdown 文档。

## 项目亮点

- 基于 AI 自动生成结构化测试用例
- 支持功能测试、边界值测试、异常测试等类型
- 采用流式输出，生成过程中可实时看到结果
- 支持按关键字和优先级筛选测试用例
- 支持导出 Excel 和 Markdown 两种格式
- 纯前端实现，启动成本低，便于演示与交付

## 运行方式

### 1. 环境要求

- Node.js 18 及以上
- npm 9 及以上

### 2. 进入项目目录

```bash
cd 01-交付作品/ai-test-case-generator
```

### 3. 安装依赖

```bash
npm install
```

### 4. 配置环境变量

在 `01-交付作品/ai-test-case-generator/.env` 中配置以下内容：

```env
VITE_DEEPSEEK_API_KEY=你的实际API密钥
VITE_DEEPSEEK_MODEL=deepseek-ai/DeepSeek-V3
```

说明：

- `VITE_DEEPSEEK_API_KEY` 为必填项
- 未配置或配置错误时，页面会提示 AI 生成失败，并回退展示内置 Mock 数据

### 5. 启动项目

```bash
npm run dev
```

启动后默认访问：

```text
http://localhost:5173
```

### 6. 构建生产包

```bash
npm run build
```

## 使用流程

1. 打开首页，点击“开始生成”
2. 输入需求描述，例如登录、注册、订单提交等业务场景
3. 选择测试类型和期望生成数量
4. 点击“生成用例”，等待 AI 流式返回结果
5. 在结果区按关键字或优先级筛选
6. 导出为 Excel 或 Markdown 文件

## 主要功能

### AI 用例生成

根据用户输入的自然语言需求，结合测试思路生成结构化测试用例，字段包括：

- 用例编号
- 用例标题
- 所属模块
- 优先级
- 测试类型
- 前置条件
- 测试步骤
- 预期结果
- 测试数据
- 自动化建议

### 流式输出体验

项目通过流式接口逐步接收模型返回内容，避免一次性长时间等待，提升交互体验。

### 条件筛选

生成完成后支持：

- 按标题或模块关键字搜索
- 按优先级筛选
- 按当前筛选结果导出

### 多格式导出

- Excel：适合测试执行、归档和统计
- Markdown：适合粘贴到 Jira、Confluence、飞书文档等平台

## 实现说明

### 前端架构

项目采用纯前端架构，不依赖独立后端服务。开发环境下通过 Vite 代理将 `/api` 请求转发到模型服务接口，减少浏览器跨域问题。

### AI 接入方式

- 通过环境变量读取模型 API Key
- 使用流式响应接收生成结果
- 前端在接收过程中尝试解析局部 JSON
- 最终完成后再进行一次完整解析，保证结果尽量完整

### 容错策略

当 API 调用失败、网络异常或配置缺失时，系统会自动降级展示预置的 Mock 测试用例，保证演示过程可继续进行。
